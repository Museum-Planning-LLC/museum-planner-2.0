# museum-planner-2.0

Source and documentation for **Museum Planner** (**museumplanner.org**): positioning, voice, static homepage, and repo hygiene. It sits alongside **Museum Planning, LLC** (**museumplanning.com**) as the **writing and archive** side of the practice—not the commercial checkout for engagements.

**Repo home:** **Museum Planning LLC** on GitHub — [`Museum-Planning-LLC/museum-planner-2.0`](https://github.com/Museum-Planning-LLC/museum-planner-2.0). **Not** the Walhimer Studio artwork site (**walhimer.github.io**).

## Files in this repo

Paths like `/archive` match WordPress-style URLs for eventual parity. |

## Quick positioning

- **museumplanning.com** — **Museum Planning, LLC**: hire for feasibility, master planning, historic buildings, opening day.
- **museumplanner.org** — **Museum Planner**: arguments, archive, books, public thinking; **Hire Mark** and practice links point back to the firm.
- **Visuals** — Homepage = dark editorial system in `index.html`; see `STYLE-GUIDE.md` for tokens and fonts.

## GitHub Pages

To serve `index.html` at `https://museum-planning-llc.github.io/museum-planner-2.0/`, enable **Settings → Pages** (branch `main`, root `/`). Until then, open the file locally or deploy elsewhere.
